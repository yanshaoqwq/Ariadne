# Agent Harness for Novel Writing - 实施计划

本计划为老版，仅作备份，请看“项目总计划-架构版(禁止删除).md”

## Context（背景与目标）

这是一个面向小说创作的可视化 agent harness 系统。核心目标是**通过多智能体协作和版本控制，增强 LLM 的小说创作能力**。

### 核心需求
- **可视化节点编辑器**：支持创建智能体节点、连线定义数据流，允许事件驱动的循环连接
- **文档编辑能力**：编辑适合小说创作的文本文档（Markdown、纯文本、章节大纲等）
- **版本控制**：Git 集成，追踪智能体对文档的每次修改
- **Skill 系统**：支持装载、管理和执行自定义 skill（类似 Claude Code 的 skill 机制）
- **上下文工程系统**：处理百万字级别的长篇小说，使用专业的上下文管理策略（RAG、分层索引、语义检索）而非简单总结
- **智能体类型**：
  - LLM 对话智能体（用于生成、改写、扩写内容）
  - 文档处理智能体（解析、转换、分析文档结构）
  - 搜索智能体（文献检索、参考资料查找）
  - Skill 智能体（执行装载的 skill）

### 技术选型（已确认）
- **技术栈**: Tauri + React + TypeScript
- **数据流模型**: 事件驱动网络（支持循环和反馈回路）
- **后端语言**: Rust（Tauri 原生）
- **前端框架**: React 18 + TypeScript
- **状态管理**: Zustand

### 2.0 Skill 系统设计（新增）

Skill 是可复用的、参数化的功能单元，类似于 Claude Code 的 skill 机制。用户可以装载、配置和在工作流中使用 skill。

#### Skill 定义格式

采用类似 Claude Code 的 YAML/JSON 格式：

```yaml

version: 1.0.0
description: 总结小说章节的核心内容
author: user

input_schema:
  type: object
  properties:
    chapter_text:
      type: string
      description: 章节原文
    max_length:
      type: number
      description: 摘要最大字数
      default: 200

output_schema:
  type: object
  properties:
    summary:
      type: string
      description: 章节摘要
    key_events:
      type: array
      items:
        type: string
      description: 关键事件列表

  type: llm  # 可选值: llm, script, http
  config:
    model: claude-opus-4
    prompt_template: |

      章节内容：
      {{chapter_text}}
      
      请提取关键事件列表（3-5个要点）。
```

```rust
// src-tauri/src/services/skill_loader.rs
use std::path::PathBuf;
use std::fs;

pub struct SkillLoader {
    global_skill_dir: PathBuf,
    project_skill_dir: PathBuf,
}

impl SkillLoader {
    pub fn new(project_root: PathBuf) -> Self {
        let global_skill_dir = dirs::config_dir()
            .unwrap()
            .join("literature-agent")
            .join("skills");
        let project_skill_dir = project_root.join(".skills");
        
        Self { global_skill_dir, project_skill_dir }
    }

    pub fn load_all_skills(&self) -> Result<Vec<Skill>, String> {
        let mut skills = Vec::new();
        
        // 加载全局 skills
        if let Ok(entries) = fs::read_dir(&self.global_skill_dir) {
            for entry in entries.flatten() {
                if let Some(skill) = self.load_skill_file(&entry.path()) {
                    skills.push(skill);
                }
            }
        }
        
        // 加载项目 skills（覆盖全局同名 skill）
        if let Ok(entries) = fs::read_dir(&self.project_skill_dir) {
            for entry in entries.flatten() {
                if let Some(skill) = self.load_skill_file(&entry.path()) {
                    // 移除同名的全局 skill
                    skills.retain(|s| s.name != skill.name);
                    skills.push(skill);
                }
            }
        }
        
        Ok(skills)
    }

    fn load_skill_file(&self, path: &Path) -> Option<Skill> {
        if path.extension()? == "yaml" || path.extension()? == "yml" {
            let content = fs::read_to_string(path).ok()?;
            serde_yaml::from_str(&content).ok()
        } else if path.extension()? == "json" {
            let content = fs::read_to_string(path).ok()?;
            serde_json::from_str(&content).ok()
        } else {
            None
        }
    }

    pub fn install_skill(&self, skill_path: PathBuf, global: bool) -> Result<(), String> {
        let target_dir = if global {
            &self.global_skill_dir
        } else {
            &self.project_skill_dir
        };
        
        fs::create_dir_all(target_dir).map_err(|e| e.to_string())?;
        
        let file_name = skill_path.file_name().unwrap();
        let target_path = target_dir.join(file_name);
        
        fs::copy(skill_path, target_path).map_err(|e| e.to_string())?;
        Ok(())
    }
}
```

```rust
#[tauri::command]
pub async fn list_skills(project_root: String) -> Result<Vec<Skill>, String> {

    loader.load_all_skills()
}

#[tauri::command]
pub async fn install_skill(
    skill_path: String,
    project_root: String,
    global: bool,
) -> Result<(), String> {
    let loader = SkillLoader::new(PathBuf::from(project_root));
    loader.install_skill(PathBuf::from(skill_path), global)
}

#[tauri::command]
pub async fn execute_skill(
    skill_name: String,
    input_data: serde_json::Value,
    project_root: String,
) -> Result<serde_json::Value, String> {
    let loader = SkillLoader::new(PathBuf::from(project_root));
    let skills = loader.load_all_skills()?;
    
    let skill = skills.iter()
        .find(|s| s.name == skill_name)
        .ok_or("Skill not found")?;
    
    execute_skill_internal(skill, input_data).await
}

async fn execute_skill_internal(
    skill: &Skill,
    input_data: serde_json::Value,
) -> Result<serde_json::Value, String> {
    match &skill.executor {
        SkillExecutor::LLM { config } => {
            // 渲染 prompt template
            let prompt = render_template(&config.prompt_template, &input_data)?;
            
            // 调用 LLM
            let llm_client = LLMClient::new();
            let response = llm_client.complete(prompt, config).await?;
            
            Ok(serde_json::json!({ "result": response }))
        }
        SkillExecutor::Script { language, code } => {
            // 执行脚本（需要集成脚本引擎）
            execute_script(language, code, &input_data).await
        }
        SkillExecutor::HTTP { url, method, headers } => {
            // 发送 HTTP 请求
            execute_http_request(url, method, headers, &input_data).await
        }
    }
}

fn render_template(template: &str, data: &serde_json::Value) -> Result<String, String> {
    // 简单的模板渲染：替换 {{key}} 为 data[key]
    let mut result = template.to_string();
    
    if let Some(obj) = data.as_object() {
        for (key, value) in obj {
            let placeholder = format!("{{{{{}}}}}", key);
            let replacement = match value {
                serde_json::Value::String(s) => s.clone(),
                _ => value.to_string(),
            };
            result = result.replace(&placeholder, &replacement);
        }
    }
    
    Ok(result)
}
```

#### Skill 在工作流中的使用

在工作流执行引擎中，SkillNode 会：
1. 获取上游节点的输出作为输入
2. 调用 `execute_skill` 命令执行 skill
3. 将结果传递给下游节点

```rust
// src-tauri/src/services/workflow_engine.rs
async fn execute_skill_node(&mut self, node: &Node) -> Result<NodeOutput> {
    let skill_name = node.config["skillName"].as_str().unwrap();
    let input_params = &node.config["inputParams"];
    
    // 合并上游输出和配置参数
    let input_data = self.merge_inputs(node.id, input_params)?;
    
    // 执行 skill
    let result = execute_skill(skill_name.to_string(), input_data, self.project_root.clone()).await?;
    
    Ok(NodeOutput {
        node_id: node.id.clone(),
        data: result,
    })
}
```

```rust
// src-tauri/src/services/workflow_engine.rs

async fn execute_llm_node(&mut self, node: &Node) -> Result<NodeOutput> {
    let rag_tools = RAGTools::new(self.knowledge_base.clone());
    let tool_defs = rag_tools.tool_definitions();
    
    let mut messages = self.build_messages(node)?;
    
    // 构建 Claude 请求（带 tools）
    let request = ClaudeRequest {
        model: node.config["model"].as_str().unwrap_or("claude-sonnet-4-20250514"),
        max_tokens: 4096,
        messages,
        tools: tool_defs,  // 注入 RAG 工具
    };
    
    // 循环处理 tool_use（LLM 可能多次调用工具）
    loop {
        let response = self.llm_client.complete(&request).await?;
        
        match response.stop_reason.as_str() {
            "tool_use" => {
                // LLM 要求调用工具
                for tool_call in response.tool_calls() {
                    let result = rag_tools.execute_tool(
                        &tool_call.name,
                        tool_call.input.clone(),
                    ).await?;
                    
                    // 将工具结果追加到 messages，继续对话
                    messages.push(Message::tool_result(tool_call.id, result));
                }
            }
            "end_turn" => {
                // LLM 完成生成
                return Ok(NodeOutput {
                    node_id: node.id.clone(),
                    data: json!({ "text": response.text() }),
                });
            }
            _ => break,
        }
    }
}
```

```rust
pub struct CheckpointConfig {
    pub enabled: bool,
    pub auto_save: bool,
    pub message_template: Option<String>,
    pub include_input: bool,
    pub include_output: bool,
}

#[tauri::command]
pub async fn git_init(repo_path: String) -> Result<(), String> {
    Repository::init(&repo_path).map_err(|e| e.to_string())?;
    Ok(())
}

#[tauri::command]
pub async fn create_checkpoint(
    repo_path: String,
    node_id: String,
    node_name: String,
    input_summary: Option<String>,
    output_summary: Option<String>,
    config: CheckpointConfig,
) -> Result<String, String> {
    if !config.enabled {
        return Ok("Checkpoint disabled".to_string());
    }
    
    let repo = Repository::open(&repo_path).map_err(|e| e.to_string())?;
    
    // 构建提交消息
    let mut message = config.message_template.unwrap_or_else(|| {
        format!("[Node: {}] Checkpoint", node_name)
    });
    
    if config.include_input {
        if let Some(input) = input_summary {
            message.push_str(&format!("\nInput: {}", truncate(&input, 100)));
        }
    }
    
    if config.include_output {
        if let Some(output) = output_summary {
            message.push_str(&format!("\nOutput: {}", truncate(&output, 100)));
        }
    }
    
    message.push_str(&format!("\nNode ID: {}", node_id));
    message.push_str(&format!("\nTimestamp: {}", chrono::Utc::now().to_rfc3339()));
    
    // 执行 Git commit
    let mut index = repo.index().map_err(|e| e.to_string())?;
    index.add_all(["."].iter(), git2::IndexAddOption::DEFAULT, None)
        .map_err(|e| e.to_string())?;
    index.write().map_err(|e| e.to_string())?;
    
    let tree_id = index.write_tree().map_err(|e| e.to_string())?;
    let tree = repo.find_tree(tree_id).map_err(|e| e.to_string())?;
    
    let signature = repo.signature().map_err(|e| e.to_string())?;
    let parent_commit = repo.head()
        .ok()
        .and_then(|h| h.target())
        .and_then(|oid| repo.find_commit(oid).ok());
    
    let parents = match parent_commit.as_ref() {
        Some(c) => vec![c],
        None => vec![],
    };
    
    let commit_oid = repo.commit(
        Some("HEAD"),
        &signature,
        &signature,
        &message,
        &tree,
        &parents,
    ).map_err(|e| e.to_string())?;
    
    Ok(commit_oid.to_string())
}

#[tauri::command]
pub async fn git_history(
    repo_path: String,
    limit: usize,
) -> Result<Vec<CommitInfo>, String> {
    let repo = Repository::open(&repo_path).map_err(|e| e.to_string())?;
    let mut revwalk = repo.revwalk().map_err(|e| e.to_string())?;
    revwalk.push_head().map_err(|e| e.to_string())?;
    
    let mut commits = Vec::new();
    for oid in revwalk.take(limit) {
        let oid = oid.map_err(|e| e.to_string())?;
        let commit = repo.find_commit(oid).map_err(|e| e.to_string())?;
        
        commits.push(CommitInfo {
            hash: oid.to_string(),
            message: commit.message().unwrap_or("").to_string(),
            author: commit.author().name().unwrap_or("").to_string(),
            timestamp: commit.time().seconds(),
        });
    }
    
    Ok(commits)
}

#[tauri::command]
pub async fn git_rollback(
    repo_path: String,
    commit_hash: String,
) -> Result<(), String> {
    let repo = Repository::open(&repo_path).map_err(|e| e.to_string())?;
    let oid = git2::Oid::from_str(&commit_hash).map_err(|e| e.to_string())?;
    let commit = repo.find_commit(oid).map_err(|e| e.to_string())?;
    
    repo.reset(commit.as_object(), git2::ResetType::Hard, None)
        .map_err(|e| e.to_string())?;
    
    Ok(())
}
```

### 3.2 执行策略

1. **拓扑排序 + 事件队列**
   - 对于无环部分，使用拓扑排序顺序执行
   - 对于有环部分（event 类型边），使用事件队列异步触发

2. **状态管理**
   - 每个节点维护输入/输出状态
   - 检测状态变化以触发事件驱动节点

3. **并发执行**
   - 独立节点并行执行
   - 使用 Tokio 异步运行时

```rust
pub struct WorkflowEngine {
    workflow: Workflow,
    node_states: HashMap<String, NodeState>,
    event_queue: Arc<Mutex<VecDeque<Event>>>,
}

impl WorkflowEngine {
    pub async fn execute(&mut self) -> Result<ExecutionResult> {
        // 1. 拓扑排序
        let sorted = self.topological_sort()?;
        
        // 2. 执行节点
        for node_id in sorted {
            self.execute_node(&node_id).await?;
        }
        
        // 3. 处理事件队列
        while let Some(event) = self.event_queue.lock().await.pop_front() {
            self.handle_event(event).await?;
        }
        
        Ok(ExecutionResult::default())
    }

    async fn execute_node(&mut self, node_id: &str) -> Result<()> {
        let node = self.workflow.get_node(node_id)?;
        
        match node.type_ {
            NodeType::LLM => self.execute_llm_node(node).await,
            NodeType::Document => self.execute_document_node(node).await,
            NodeType::Skill => self.execute_skill_node(node).await,
            // ...
        }
    }
}
```

## 关键技术决策记录

### 1. 为什么选择 Qdrant 嵌入式而非其他方案？

**对比:**
- **Qdrant 嵌入式**: Rust 原生，无需独立进程，适合桌面应用，性能优秀
- **ChromaDB**: Python 为主，虽有 Rust 绑定但不如原生流畅
- **Milvus**: 需要独立服务，重量级，不适合桌面应用
- **自建 HNSW + SQLite**: 实现复杂度高，维护成本大

**结论**: Qdrant 嵌入式最适合 Tauri 桌面应用场景。

### 2. 为什么用 tool_use 而不是自动 RAG？

**自动 RAG 方案**: 每次 LLM 调用前，系统自动检索相关内容并塞进 prompt。

**问题**:
- LLM 无法控制检索粒度和范围
- 可能检索到不相关内容，浪费 token
- 缺乏灵活性（无法根据生成需求动态调整检索策略）

**tool_use 方案**: LLM 主动决定何时、如何检索。

**优势**:
- LLM 可以根据上下文智能选择工具
- 支持多轮检索，逐步细化查询
- 更符合 agent 的工作模式

### 3. 为什么同时保留结构化数据（SQLite）和向量数据库？

**向量数据库**: 适合语义相似度搜索，但不适合精确查询（如"角色X的生日是哪天？"）

**SQLite**: 适合结构化数据的精确查询、关系查询

**混合策略**: 
- 全文内容 → 向量化 → Qdrant（语义搜索）
- 角色档案、世界观设定、时间线 → SQLite（精确查询）
- LLM 工具同时包含两类查询能力
