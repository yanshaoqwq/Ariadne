# Ariadne

**为长篇小说而生的 AI 创作工作台。**

Ariadne 把大纲规划、设定管理、正文写作、审稿返修、章节总结、全文检索和版本存档放进同一个桌面应用。你可以像搭建流程图一样安排多个 AI 的分工，同时保留对正文、预算、权限和最终决定的控制。

它不只是一个“帮我续写”的聊天框，而是一套面向长篇创作的完整工作环境：知道作品写到了哪里，记得人物和伏笔如何变化，能说明一段文字为什么被修改，也能在结果不满意时回到之前的存档点。

> **Ariadne 目前尚未正式发布。**

## Ariadne 能为你做什么

### 把创作过程变成可运行的工作流

长篇小说很少是“一次提示词生成全文”。在 Ariadne 中，你可以让不同 AI 分别承担不同角色：

- 总览者规划全书主旨、阶段和整体方向。
- 设计师细化当前阶段的结构与叙事目标。
- 大纲 Agent 规划下一章，并登记人物、关系和伏笔变化。
- 细节 Agent 补充环境、心理、动作、考据等内容。
- Writer 根据大纲、前文和细节写出正文。
- Critic 从不同角度提出审稿意见。
- Prudent 判断哪些意见合理，并形成返修目标。
- Polisher 按审稿结论修改正文。
- Summarizer 更新故事段、事件、章节和阶段总结。

这些步骤可以在可视化画布中自由组合。你可以增加条件、限制循环次数、设置断点，也可以要求某一步必须经过人工确认后才能继续。

### 为长篇项目保留真正有用的记忆

Ariadne 不依赖把整本书一次性塞进模型。它会按不同粒度管理创作信息：

- 故事段与事件。
- 章节总结、阶段总结和全书方向。
- 人物档案、性格变化和人物关系。
- 已埋下、待回收和已回收的伏笔。
- 世界观、主题、长期设定和作者意图。
- 正文、大纲、知识条目与生成结果之间的来源引用。

AI 在规划或写作时，可以检索真正相关的内容，而不是只依赖最近几轮聊天记录。大段正文通过文档引用按需读取，避免在多个节点之间反复复制全文。

### 让 AI 写作可审阅、可暂停、可恢复

每次重要写回都可以先生成差异和确认项。你可以查看修改内容、批准、拒绝或要求重新处理。

- **Pause**：暂停当前工作流，保留运行现场，之后继续。
- **Stop**：终止本次运行，保留已经完成的结果和记录。
- **Resume**：从保存的运行状态继续，而不是重新执行全部步骤。
- **断点**：在指定节点停下，检查输入、输出和上下文。
- **运行日志**：查看节点、模型、工具、成本、确认项和错误信息。

对于已经反复验证的流程，可以开启 Auto Mode；预算、文件范围、网络访问、密钥和工具权限仍由独立规则控制。

### 把作品、知识和版本放在同一个项目里

每个 Ariadne 项目对应一个系列作品。正文、大纲、工作流、提示词模板、知识、运行记录和导出结果都围绕这个项目组织。

你可以：

- 创建或打开本地小说项目。
- 导入已有章节并维护作品目录。
- 在阅读与修改模式之间切换。
- 对选中的文字使用快捷 AI 改写。
- 管理项目记忆和长期设定。
- 合并导出 Markdown、EPUB 或 PDF。
- 在重要修改前创建手动存档。
- 查看自动 checkpoint 和版本历史。
- 从旧版本创建新分支恢复，不覆盖当前创作线。

## 一套完整的长篇创作流程

```text
全书规划
   ↓
阶段设计
   ↓
章节大纲 ──→ 登记人物 / 关系 / 伏笔变化
   ↓
细节设计与资料检索
   ↓
正文写作
   ↓
多角度审稿
   ↓
意见判断与返修
   ↓
人工确认或 Auto Mode 审计
   ↓
写回正文 + Git 存档
   ↓
故事段 / 事件 / 章节 / 阶段总结
   ↓
进入下一章
```

这套流程不是固定模板。你可以删减角色、替换模型、加入自己的 PromptTemplate，或者把一套成熟流程保存为 Workflow，供新项目复用。

## 主要页面

### 工作空间

工作空间是 Ariadne 的流程编排与运行中心。

- 从节点库添加写作、审稿、检索、条件、审批、循环和导出节点。
- 通过控制线、数据线和通信线表达执行顺序、数据传递与审稿交流。
- 为节点选择模型、提示词、预算、超时、工具和断点。
- 查看节点运行状态、输入输出、确认项和错误。
- 使用缩放、小地图和框选管理大型工作流。
- 运行完整工作流，或从指定起始节点开始。

### 作品

作品页用于阅读、编辑和管理正文。

- 按系列、阶段、卷和章节浏览作品。
- 导入外部稿件或创建新章节。
- 查看字数、版本和保存状态。
- 使用快捷改写生成修改建议并检查差异。
- 从正文定位到相关大纲和项目知识。
- 多选章节并按指定顺序合并导出。
- 随时打开项目 AI，讨论当前章节或引用项目内容。

### 项目 AI

项目 AI 是理解整个项目的总览助手。它可以结合聊天历史和项目记忆，并通过 `@` 引用指定对象：

- `@章节` 或 `@文档`。
- `@节点` 的输入与输出。
- `@确认项` 的差异和处理结果。
- `@知识` 中的人物、事件、关系和伏笔。
- `@artifact` 中的生成或导出结果。

你也可以把某个工作流作为项目 AI 的工具，让它在对话中调用已经配置好的创作流程。

### Git 与版本存档

版本页用面向作者的“存档”概念管理 Git：

- 创建章节完成、结构调整或批量润色前的手动存档。
- 查看工作流自动生成的节点 checkpoint。
- 浏览分支、提交关系、摘要和差异。
- 从历史存档创建恢复分支，保留当前版本。
- 在回档后重新绑定运行状态并重建需要更新的索引。

### 运行日志

运行日志集中展示：

- 工作流、运行和节点状态。
- 模型请求与工具调用。
- 成本、预算和 token 使用。
- 确认项与 Auto Mode 审计结果。
- 暂停、恢复、重试和错误原因。
- 检索、索引、Provider 和后端诊断信息。

你可以按级别和关键词筛选，用它回答“哪一步失败了”“为什么暂停”“改动从哪里产生”等问题。

### 模板市场

模板市场用于搜索和导入可复用的 Workflow。导入后的模板会展开为普通节点和连线，你可以自由修改，不会被远端模板自动覆盖。

如果模板需要网络、文件或工具权限，Ariadne 会在导入前展示权限声明。模板发布新版本时，只提示可以升级，不会静默改变现有工作流。

### 配置

配置页集中管理：

- Provider、API Key、模型与默认模型。
- Embedding、Reranker 和搜索服务。
- 节点类型的默认模型、预算和超时。
- Auto Mode、确认策略和成本限制。
- 文件、网络、工具和 WASM 权限。
- RAG、索引、Git 和运行参数。
- 浅色、深色、自定义主题和界面颜色。
- 语言、诊断和项目级偏好。

## 模型与服务

Ariadne 通过统一 Provider 层连接不同模型服务，支持：

- OpenAI API 格式与兼容服务。
- Anthropic API 格式。
- Gemini API 格式。
- 本地或自托管模型端点。
- 独立的 Embedding、Reranker 和 Search Provider。

模型配置与工作流分离。你可以为 Writer、Critic、Summarizer 等节点选择不同模型，也可以在不修改流程结构的情况下更换 Provider。

API Key 通过密钥存储保存，项目配置只引用密钥编号，不需要把明文密钥写进工作流或正文目录。

## 成本、权限与安全

Ariadne 把自动化建立在明确边界上：

- 设置单次、每日、每月和 Auto Mode 预授权预算。
- 高成本调用可以暂停并等待确认。
- 网络访问按 Provider、Search 和 HTTP 工具分别授权。
- 文件读写限制在允许的项目路径中。
- 路径穿越和符号链接逃逸会被拒绝。
- WASM 执行器受到能力、时间、内存和输出限制。
- 运行日志不会保存完整认证头或明文密钥。
- 外部模板导入前展示权限需求。

Auto Mode 只减少普通确认，不会关闭这些硬限制。

## 长文本与检索

Ariadne 面向百万字级长篇项目设计。检索系统可以组合：

- 关键词与全文检索。
- 向量语义检索。
- 结构化知识筛选。
- 章节、阶段和全书摘要。
- 冷热索引与按需召回。
- Reranker 重排。
- 来源范围、人物、事件、章节和时间等过滤条件。

正文是事实源，索引可以重新构建。即使检索索引损坏，也不会把索引当作作品的唯一副本。

## 开始使用

1. 打开 Ariadne，创建一个新项目或选择已有项目目录。
2. 在“配置 → 模型”中添加 Provider、模型和 API Key。
3. 设置默认写作模型、检索模型、预算和权限。
4. 在“作品”中导入已有章节，或创建第一章。
5. 在“工作空间”中导入模板或搭建自己的创作工作流。
6. 第一次运行时保留人工确认，逐项检查节点输出和正文差异。
7. 流程稳定后，再按需要开启 Auto Mode。
8. 在重要章节完成或大规模修改前创建 Git 存档。

## 安装与运行

正式安装包将在 Release 页面提供。当前可从源码运行。

### 环境要求

- Rust `1.78` 或更新版本。
- .NET SDK `10.0`。
- Git。

### 从源码启动

```bash
git clone git@github.com:yanshaoqwq/Ariadne.git
cd Ariadne

dotnet restore desktop/Ariadne.slnx
./desktop/run-ui.sh run
```

也可以手动构建后端和桌面端：

```bash
cargo build --workspace --bins
dotnet build desktop/Ariadne.slnx

ARIADNE_BACKEND_IPC=target/debug/ariadne-ipc \
  dotnet run --project desktop/Ariadne.Desktop
```

## 给其它程序和 Agent 调用

Ariadne 的同一套工作流能力也可以通过 CLI、JSON-line IPC 或本地 REST 服务调用。

### CLI

```bash
target/debug/ariadne tools list --project /path/to/project --json

target/debug/ariadne workflow run \
  --project /path/to/project \
  --workflow default \
  --start start-main \
  --json

target/debug/ariadne workflow events \
  --project /path/to/project \
  --workflow default \
  --run run-xxx \
  --after 0 \
  --limit 50 \
  --json
```

### JSON-line IPC

```bash
target/debug/ariadne-ipc stdio
```

### 本地 REST

```bash
target/debug/ariadne-server \
  --project /path/to/project \
  --bind 127.0.0.1:4817 \
  --token '<strong-random-token>'
```

除 `/health` 外，请求需要携带：

```http
Authorization: Bearer <strong-random-token>
```

REST 服务默认用于本机单项目集成，不应在没有额外网络防护的情况下直接暴露到公网。

## 项目文件

```text
my-novel/
├── .config/        # 模型、权限、RAG、工作流、Git 与 Auto Mode 配置
├── .runtime/       # 运行状态、日志和 checkpoint
├── .indexes/       # 可重建的检索索引
├── .knowledge/     # 创作知识与元数据
├── documents/      # 正文、总纲、阶段大纲和章节大纲
├── workflows/      # 可编辑的创作工作流
├── skills/         # PromptTemplate 与执行器资源
└── exports/        # 导出文件与生成 artifact
```

建议仍然对重要项目保留独立备份。Git 存档用于追踪创作版本，不应替代磁盘、云盘或其它离线备份。

## 显示语言

Ariadne 当前以中文作为正式界面语言。语言包采用同名键覆盖机制，未翻译的文本自动回退到中文；后续语言包可以在不修改业务代码的情况下加入。

## 名字的含义

在希腊神话中，Ariadne 的线帮助人走出迷宫。

这个名字也代表本项目的目标：在不断增长的正文、设定、人物、事件、伏笔、模型输出和版本历史之间，始终保留一条能找到来源、理解过程并回到正确位置的线。
